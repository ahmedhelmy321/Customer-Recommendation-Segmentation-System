## 🚀 Quick Start

### 1. Install dependencies
```bash
pip install -r requirements.txt
```

### 2. Add the dataset
Place `OnlineRetail.xlsx` in `data/raw/`.
Download from: https://archive.ics.uci.edu/ml/datasets/Online+Retail

### 3. Run the notebook
Open and run all cells in:
```
notebooks/Recommendation_System_Pipeline.ipynb
```
This generates:
- `data/processed/customer_offers.csv`
- `data/processed/customer_segments.csv`

*(Optional)* Save models by adding to the notebook:
```python
import joblib
joblib.dump(kmeans, 'models/kmeans_model.pkl')
joblib.dump(scaler, 'models/scaler.pkl')
autoencoder.save('models/autoencoder.keras')
```

### 4. Launch the Streamlit app
```bash
streamlit run app.py
```

## 🧠 ML Pipeline Overview

### Step 1 — Data Preprocessing
- **541K raw transactions** → **~270K clean rows**
- Drop missing CustomerID, cancellations (InvoiceNo starts with 'C'),
  negative quantities, zero prices, noise descriptions
- IQR upper-bound outlier removal on Quantity, UnitPrice, TotalPrice

### Step 2 — Feature Engineering
**RFM + Extended Features** (1 row per customer):
| Feature | Description |
|---|---|
| Recency | Days since last order |
| Frequency | Number of unique invoices |
| Monetary | Total spend |
| AvgOrderValue | Monetary / Frequency |
| UniqueProducts | Distinct StockCodes bought |
| AvgItemsPerOrder | Total items / orders |

All skewed features log-transformed (`log1p`) then StandardScaled.

**Customer × Product Matrix**: log1p(quantity) per (customer, product) pair.
Products with fewer than 20 buyers are excluded (cold-start filter).

### Step 3 — Customer Segmentation

| Model | Purpose |
|---|---|
| **KMeans (K=4)** | Hard segment assignment: Champions / Loyal / At-Risk / Dormant |
| **GMM** | Soft probabilistic membership. Borderline customers (confidence < 0.6) get GMM-blended offers |

Cluster labels assigned automatically based on RFM profile logic.

### Step 4 — Recommendation Engine (AutoEncoder)

Architecture: `input → 256 → 128 → 64 → 128 → 256 → output`

- Input: customer purchase vector (log1p quantities, MinMaxScaled)
- Bottleneck (64D) learns compressed purchase pattern embedding
- Decoder reconstructs ALL product scores — non-zero scores on unvisited products = recommendations
- Already-purchased items are masked out → **novelty = 100% by design**

### Step 5 — Offer Generation

| Segment | Discount | Strategy |
|---|---|---|
| 👑 Champions | 5% | VIP Early Access |
| 🟢 Loyal | 10% | Loyalty Reward |
| ⚠️ At-Risk | 15% | Win-Back Campaign |
| 💤 Dormant | 20% | Re-Engagement Offer |
